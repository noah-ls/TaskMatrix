# Security Policy

## Supported Versions

Security fixes are considered for the latest code on `main` and the latest
published release, if releases are available.

## Reporting a Vulnerability

Please do not open a public issue for security vulnerabilities.

Use GitHub's private vulnerability reporting feature if it is enabled for the
repository. If it is not enabled, open a minimal public issue asking for a
maintainer security contact without disclosing technical details.

Include:

- Affected version or commit.
- Steps to reproduce.
- Expected and actual behavior.
- Impact assessment.
- Any known workaround.

Maintainers will acknowledge valid reports as soon as practical and coordinate
fixes before public disclosure.
